# 项目建设阶段 · 周报撰写素材（草拟）

> 用途：整理**本周进度、上线与培训、数据迁移**相关表述，便于填写个人周报。  
> 下表为**示例概括**，请按你真实工时与日期替换；「最近五周」可按自然周调整。

---

## 一、本周当前进度（可直接摘一句到周报「本周完成」）

- 推进 **IWS 应用** 上线前事项：环境变量与 Docker 部署配置复核、Supabase 公网/内网访问拆分验证。  
- 完成/跟进 **用户与部门** 相关改造：部门负责人改为「部门 LD 角色」维护、用户创建改为「先写业务库再邀请」及批量导入不自动发信。  
- 输出 **培训与交付物**：最终用户《系统周报介绍》、样例 PRD/技术方案/Next.js 说明/检索方案等文档，供评审与知识沉淀。  
- （可选）联调 **文件上传与检索** 链路，处理 Storage 限额与头像/公网 URL 等问题。

---

## 二、项目上线与培训（周报里可写成「计划 / 进行中 / 已完成」）

| 事项 | 说明（概括） |
|------|----------------|
| 上线检查 | 生产环境变量、`SITE_URL`、Supabase 密钥与 Storage 全局限额确认；镜像构建与 compose 发布流程走通。 |
| 试运行 | 选定试点部门或项目，收集界面与流程问题清单。 |
| 培训 | 面向最终用户讲解：**周报填写与审批**、**项目文件上传与检索**；面向管理员讲解：**用户/部门/项目维护**与**邀请开通账号**。 |
| 材料 | 发放《系统周报功能说明》链接或 PDF；管理员保留操作备忘（用户邀请、导入用户不发邀请后的补邀）。 |

---

## 三、数据迁移（周报里可单独一小节）

| 事项 | 说明（概括） |
|------|----------------|
| 范围 | 历史用户/部门/项目/成员关系是否迁入；是否仅迁移「进行中」项目。 |
| 用户与登录 | 批量导入 `public.users` 后，统一或分批 **发送邀请** 绑定 Auth；历史头像 URL 若曾为内网地址，需批量改为「仅路径」或公网可访问形式。 |
| 文件 | 若存在历史文件在其他盘或系统，约定迁移窗口与校验方式（只读校验、抽检打开）。 |
| 割接 | 冻结写操作时间点、迁移后冒烟清单（登录、周报一条、文件上传、检索一条）。 |

---

## 四、最近五周 · 工作内容 & 工作计划（示例）

> 每周分两块：**工作内容**（本周已完成）、**工作计划**（下周打算）；各 3～5 条为宜。以下为**示例概括**，可按真实周次替换。

### 第 1 周（示例：基础架构与账号）

**工作内容**

1. 搭建 Next.js（App Router）工程骨架，约定 `actions` / `lib/db` / `supabase` 客户端分层。  
2. 接入 Supabase（Auth、Postgres、Storage），配置本地与 Docker 环境变量。  
3. 实现登录、会话与基础布局；中间件统一未登录跳转。  
4. 完成用户资料与部门树只读展示雏形。  

**工作计划**

1. 梳理周报业务字段与状态机，设计「我的周报」列表与新建入口接口。  
2. 与产品确认周次规则（自然周、补填、逾期）及项目角色权限。  
3. 搭建项目与成员相关的只读查询，为周报选项目做准备。  

---

### 第 2 周（示例：周报与项目核心流程）

**工作内容**

1. 实现「我的周报」列表与新建周报：项目、周次、工作项及状态机（草稿/待审/通过/退回）。  
2. 项目经理审批流与退回原因；补填与逾期说明相关逻辑。  
3. 项目列表与成员角色校验，限制仅「进行中」项目可填周报。  
4. 联调周报与项目文件关联（上传/选自文件池）。  

**工作计划**

1. 开发管理端用户/部门/项目维护页面，并接入权限校验。  
2. 完善文件池与上传链路，对接异步处理与错误提示。  
3. 开始检索服务前端对接方案评审。  

---

### 第 3 周（示例：管理端、文件与体验）

**工作内容**

1. 管理端：用户/部门/项目的创建与编辑；部门 LD 与 `leader_id` 字段下线后的角色逻辑调整。  
2. 用户创建流程：先写入 `public.users`，再按需邀请；导入用户默认不发邀请，支持列表补邀与批量邀请。  
3. 头像与 Storage：公网地址与仅存路径策略；内网 Supabase 场景下展示 URL 修正。  
4. 文件检索前端对接检索 API；处理大文件上传与 Storage 限额提示。  

**工作计划**

1. 整理生产环境变量清单与 Docker 发布流程，起草上线检查表。  
2. 输出最终用户说明文档与内部样例 PRD/技术方案。  
3. 与业务方对齐数据迁移范围与割接窗口。  

---

### 第 4 周（示例：文档、上线准备与迁移方案）

**工作内容**

1. 编写最终用户《系统周报介绍》及样例 PRD、技术方案、Next.js 说明、检索方案等文档。  
2. 生产环境与健康检查清单；Docker 镜像与 compose 变量对齐。  
3. 数据迁移方案评审：用户批量导入、邀请节奏、历史文件与头像数据修正策略。  
4. 培训材料定稿与试点培训安排；收集问题清单并迭代。  

**工作计划**

1. 组织或参与试点培训，跟踪缺陷与需求变更。  
2. 执行数据迁移演练（或预演），修正脚本与回滚步骤。  
3. 锁定生产发布窗口，协调运维与业务冻结时间。  

---

### 第 5 周（示例：上线、迁移割接与收尾）

**工作内容**

1. 生产发布与冒烟验证（登录、周报一条、文件上传、检索一条）。  
2. 正式数据迁移或分批导入用户；按节奏发送邀请、处理开通问题。  
3. 面向管理员与最终用户的培训与答疑；归档问题清单。  
4. 试运行问题修复与监控项确认（日志、关键接口可用性）。  

**工作计划**

1. 收集上线后反馈，排期优化项与 minor bug。  
2. 与业务约定运维交接与账号权限例行检查方式。  
3. （可选）整理二期需求池（统计报表深化、移动端等）。  

---

## 五、一句话模板（填系统周报时用）

- **本周完成**：完成 ×× 功能联调 / 文档 ×× 份 / 上线检查项 ×× 项。  
- **下周计划**：数据迁移演练 / 正式培训 / 生产发布窗口确认。  
- **需协调**：业务方确认迁移冻结时间、邮件网关与邀请到达率。
