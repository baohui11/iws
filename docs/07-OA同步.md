# OA 同步

## 同步对象

OA 同步包含四类数据：

1. 部门
2. 用户
3. 项目
4. 项目角色 / 项目成员

同步顺序必须保持：

```text
部门 -> 用户 -> 项目 -> 项目角色
```

因为用户和项目依赖部门，项目角色依赖项目和用户。

## 手动同步

系统管理员可在“系统管理 / OA 同步”页面执行：

- 一键同步全部
- 同步部门
- 同步用户
- 同步项目
- 同步项目角色

部门管理员可进入页面查看同步记录，但不能执行同步。

## 定时同步

OA 定时任务运行在 `web` 容器内，默认每天 04:00 和 13:00 同步全部：

```env
OA_SYNC_SCHEDULE_ENABLED=true
OA_SYNC_SCHEDULE_CRON=0 4,13 * * *
```

当前 cron 解析器只支持每天固定小时执行的格式：

```text
分钟 小时列表 * * *
```

例如：

```env
OA_SYNC_SCHEDULE_CRON=30 3,15 * * *
```

表示每天 03:30 和 15:30 执行。

## 同步记录

同步记录写入 `oa_sync_runs`。一键同步全部会生成四条记录，分别对应部门、用户、项目和项目角色，便于定位具体失败环节。

## 数据生效

OA 是主数据来源，但不是所有同步数据都会立即参与业务：

- 部门有 `is_active`，未删除部门都会显示在管理端，业务筛选默认只显示激活部门。
- 用户有 `is_active`，同步用户默认不激活。
- 项目有 IWS 内部激活状态，默认不激活。
- 项目成员有 `is_active`，用于控制是否可填报项目周报。

## 常见失败

- OA Token 未配置或过期。
- PgREST 分页响应异常。
- 用户引用了不存在的部门。
- 项目引用了不存在的部门。
- 项目角色引用了不存在的用户或项目。
- 数据库唯一约束冲突。
