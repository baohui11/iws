# 权限模型

## 系统角色

| 角色 | 说明 |
| --- | --- |
| `admin` | 系统管理员，拥有全部数据和全部模块权限。 |
| `dept_admin` | 部门管理员，可管理权限范围内的用户和项目，查看 OA 同步记录。 |
| `dept_ld` | 部门领导，可查看本部门及下级部门数据。 |
| `bp` | BP 角色，可查看配置的数据范围内的统计和文件。 |
| `company_ld` | 公司领导，可按配置范围查看数据。 |
| `user` | 普通用户。 |

## 数据范围

数据范围由 `user_data_scopes` 维护：

- `department`：指定部门，默认包含下级部门。
- `all`：全部数据。

用户所属部门不需要额外配置，系统会自动作为基础范围参与部分个人场景判断。

## 模块权限

| 模块 | 可访问角色 |
| --- | --- |
| 主页 / 文件搜索 | 登录用户 |
| 项目周报 | 登录用户 |
| 数据统计 | admin、dept_ld、dept_admin、bp、company_ld |
| 系统管理 | admin、dept_admin |
| 数据权限 | admin |
| OA 同步执行 | admin |
| OA 同步记录 | admin、dept_admin |

## 文件权限

普通文件可被以下用户查看：

- 项目对应阶段成员。
- 文件所属部门权限范围内用户。
- 系统管理员。

保密文件可以出现在检索结果中，但没有内容权限的用户不能预览和下载。可查看保密文件内容的用户：

- 上传人
- 项目组成员
- 部门领导
- 公司领导
- 部门管理员
- 系统管理员

BP 不能查看保密文件内容，除非同时满足其他允许条件。

## 周报权限

- 项目成员只能填写自己参与阶段的周报。
- 只有进行中的项目可以填写周报。
- 销售阶段周报不审批。
- 实施阶段普通成员周报由项目经理审批。
- 项目经理自己的实施阶段周报不审批。
- 草稿、撤回、审批中周报不进入统计。
