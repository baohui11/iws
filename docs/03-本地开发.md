# 本地开发

## 环境要求

- Windows + PowerShell
- Node.js 22+
- pnpm
- Docker Desktop
- Python 3.12+（仅开发 file-worker 时需要）

## 启动基础设施

```powershell
cd E:\app\iws\docker
Copy-Item .env.example .env
docker compose up -d postgres minio gotenberg
```

## 启动 Web

```powershell
cd E:\app\iws\web
Copy-Item .env.example .env
pnpm install
pnpm dev
```

访问：

```text
http://localhost:3000
```

## 数据库

执行迁移：

```powershell
cd E:\app\iws\web
pnpm db:migrate
```

创建初始管理员：

```powershell
pnpm db:seed
```

类型检查：

```powershell
pnpm exec tsc --noEmit
```

生产构建：

```powershell
pnpm build
```

## file-worker 本地运行

```powershell
cd E:\app\iws\file-worker
Copy-Item .env.example .env
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e .
python -m iws_file_worker
```

## 常见问题

### Docker 构建时 pnpm 拒绝依赖 build scripts

项目使用 `web/pnpm-workspace.yaml` 配置 pnpm 11 的 `allowBuilds`，允许必要依赖执行构建脚本。

### 页面能访问但上传失败

检查：

- MinIO 是否启动。
- `S3_ENDPOINT` 是否为服务端可访问地址。
- `S3_PUBLIC_ENDPOINT` 是否为浏览器可访问地址。
- 桶是否已自动创建。

### 邮件发送失败

检查 SMTP 配置。公司网络或证书链异常时，本地可临时配置：

```env
SMTP_TLS_REJECT_UNAUTHORIZED=false
```

生产环境不建议关闭 TLS 校验。
