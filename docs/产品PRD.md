# 中大咨询周报文件系统 PRD

**文档版本：** v1.1

---

## 1. 产品概述

中大咨询内部管理系统，用于统一管理项目周报填写审批流程与项目成果文件，解决周报分散、文件版本混乱、缺乏知识沉淀的问题。

---

## 2. 角色体系

### 2.1 双层角色体系说明

用户同时拥有一个系统角色和零至多个项目角色（每个项目内一个），两类角色叠加生效，取权限并集。

---

### 2.2 系统角色

| 系统角色 | 说明 |
|----------|------|
| **超级管理员** | 系统最高权限，管理所有用户、项目、系统配置 |
| **部门管理员** | 管理本部门的用户与项目数据 |
| **部门 LD** | 可查看本部门所有项目的周报与文件数据 |
| **公司 LD** | 可查看全公司所有项目的周报与文件数据 |
| **普通用户** | 基础角色，权限完全由项目角色决定 |

---

### 2.3 项目角色

每个项目独立分配，同一用户在不同项目中可担任不同角色。

| 项目角色 | 数量限制 | 说明 |
|----------|----------|------|
| **项目经理（PM）** | 每项目唯一 | 审批项目成员周报；自行填写周报无需审批 |
| **项目成员** | 不限 | 填写周报，提交后由 PM 审批 |
| **项目总监** | 不限 | 可查看项目全部周报与文件；不参与审批流程 |
| **项目所属部门 LD** | 不限 | 与项目总监相同，代表部门层面的项目观察者 |

> **说明：** 项目总监与项目所属部门 LD 为只读观察角色，不参与周报填写与审批流程。

## 3. 用户与项目管理

### 3.1 用户管理

由超级管理员和部门管理员维护，支持：
- 手动创建用户，字段包括：姓名、工号、邮箱、手机、所属部门、系统角色
- 批量导入（Excel / CSV）
- 编辑用户信息、调整系统角色
- 停用用户（离职处理）：账号停用，历史数据完整保留，停用用户不再出现在可选成员列表中

### 3.2 项目管理

由超级管理员和部门管理员创建与维护，项目信息包括：

| 字段 | 说明 |
|------|------|
| 项目名称 | 必填 |
| 项目编号 | 唯一标识，可自动生成或手动填写 |
| 所属部门 | 必填 |
| 项目简介 | 项目背景与目标描述 |
| 项目标签 | 多选，如：战略咨询、IT规划、尽调等 |
| 项目状态 | 筹备中 / 进行中 / 已完结 / 已归档 |
| 开始时间 / 预计结束时间 / 实际结束时间 | |
| 合同成果文件清单 | 合同约定的交付物名称与描述列表 |

### 3.3 项目成员管理

- 为项目分配成员，指定项目角色（PM / 项目成员 / 项目总监 / 项目所属部门LD）
- 每个项目有且仅有一名 PM
- 支持调整成员的项目角色、移除成员
- 停用用户不可被新增为成员，其历史成员记录保留

## 4. 周报模块

### 4.1 周报填写

**适用角色：** PM、项目成员

用户在"我的周报"中选择项目和周次后开始填写，每份周报可添加多条工作事项。

**周次规则：**
- 以自然周（周一至周日）为单位，格式如 `2025-W03`
- 系统自动列出当前用户参与项目的可填写周次
- 支持补填历史周次，截止时间后补填须填写逾期说明

**工作事项字段：**

| 字段 | 说明 |
|------|------|
| 事项标题 | 必填 |
| 事项描述 | 工作内容详述 |
| 工时（小时） | 可选 |
| 关联文件 | 可直接上传，或从项目文件池中选择已有文件 |

**关联文件方式：**
- **直接上传：** 文件自动进入项目文件池，类型默认为成果文件
- **从文件池选择：** 在项目文件池中选择已有文件进行关联

### 4.2 周报查看

- 项目成员可查看**同项目所有人**的周报内容
- PM、项目总监、项目所属部门LD 同样可查看项目内所有周报
- 部门管理员、部门LD、公司LD、超级管理员按权限范围查看

### 4.3 审批流程

```
项目成员提交
     ↓
待审批（PM 审批）
     ↓
通过 → 已通过
退回 → 已退回（含退回原因）→ 成员修改后重新提交
```

- **PM 的周报**：填写后直接标记为已完成，无需审批
- PM 退回周报时必须填写退回原因
- 支持批量审批通过

### 4.4 逾期处理

系统设有全局周报截止时间（默认值由超级管理员配置，如每周一上午 10:00）。

- 截止时间后提交或审批的操作，系统强制要求填写**逾期说明**
- 逾期记录在统计报表中单独标注

## 5. 项目文件池

### 5.1 文件分类

| 文件类型 | 说明 |
|----------|------|
| **成果文件** | 项目交付物，可关联合同成果文件清单中的具体条目 |
| **参考文件** | 过程中使用的参考资料，可关联到某个成果文件 |

### 5.2 文件上传

- 支持单文件和批量上传（单次最多 20 个，单文件最大 200MB）
- 上传时需选择文件类型
- 上传成果文件时，可选择关联合同成果清单中的某一条目
- 上传参考文件时，可选择关联到某个成果文件
- 文件上传后进入异步处理队列，处理期间文件可正常访问与下载

### 5.3 文件池视图

- 支持列表视图与网格视图切换
- 按文件类型、上传时间、上传人、标签筛选
- 显示文件处理状态（处理中 / 已完成 / 异常）
- 重复文件、版本识别结果以标记形式展示
- 版本链：同一文件的多个版本归组展示，默认展示最新版本，可展开查看历史版本

---

## 6. 文件异步处理

文件上传完成后，由 Python 异步服务（通过 Supabase 消息队列调度）依次执行以下检测，结果写回数据库后前端实时更新标记。

| 检测类型 | 结果展示 | 用户操作 |
|----------|----------|----------|
| **重复文件检测** | 警告标记"与已有文件重复" | 可选择保留或删除 |
| **版本识别** | 提示"可能是 [原文件名] 的新版本" | 可确认版本关系或忽略 |
| **合同成果文件检测** | 提示"已匹配合同成果：[成果名称]" | 可手动修正匹配关系 |

处理状态流转：`排队中 → 处理中 → 已完成 / 处理异常`

处理异常时显示错误原因，支持手动重新触发。

## 7. 数据统计

### 7.1 统计快照生成

- 系统在每周截止时间自动生成当周统计快照
- 快照一经生成不可篡改，作为历史存档

### 7.2 统计内容

| 维度 | 内容 |
|------|------|
| 项目维度 | 各项目周报提交率、审批完成率 |
| 人员维度 | 每人周报状态（已提交 / 未提交 / 已通过 / 已退回） |
| 逾期情况 | 逾期人员列表及逾期说明 |
| 文件维度 | 当周新增文件数、成果文件完成情况 |

### 7.3 查看与导出

- 统计报表支持按周次查看历史快照
- 支持导出为 Excel 文件

**查看权限：**

| 角色 | 可查看范围 |
|------|-----------|
| 超级管理员、公司LD | 全公司所有项目 |
| 部门管理员、部门LD | 本部门所有项目 |
| PM、项目总监、项目所属部门LD | 本项目 |

## 8. 文件互动功能

### 8.1 检索

使用 Meilisearch 对文件元数据及内容进行全文检索，具体实现方案另行设计。

### 8.2 预览

使用 LibreOffice 实现 Office 文档在线预览，具体实现方案另行设计。

### 8.3 点赞

- 每人对每个文件只能点赞一次，再次点击取消
- 显示点赞总数

### 8.4 评论

- 支持对文件发表评论
- 支持一级回复（最多两级）
- 评论者本人或超级管理员可删除评论

### 8.5 收藏

- 用户可收藏有权限访问的任意文件
- 在"个人中心 - 我的收藏"中统一查看
